## 日志系统概述

该项目实现了基于Spring Boot的完整日志系统，包含**访问日志**和**操作日志**两大核心模块，通过AOP切面和MVC拦截器实现自动化记录，所有日志持久化到SQLite数据库并提供管理API。

## 核心架构

### 1. 访问日志系统（Access Log）
- **拦截器**: `AccessLogInterceptor` - 通过HandlerInterceptor自动捕获所有API请求
- **实体模型**: `AccessLog` - 存储用户、IP、URL、方法、参数、结果、错误信息等
- **记录时机**: `afterCompletion`阶段异步写入，避免影响主流程性能
- **过滤规则**: 排除登录接口(`/api/v1/auth/**`)和日志查询接口(`/api/v1/system/logs/**`)

### 2. 操作日志系统（Operation Log）
- **注解驱动**: `@OpLog` 注解标注需要记录操作的Controller方法
- **AOP切面**: `OperationLogAspect` - 使用@Around切面自动收集方法信息、参数、返回值
- **实体模型**: `OperationLog` - 支持记录修改前后数据对比
- **ThreadLocal机制**: 通过`BEFORE_DATA_HOLDER`传递修改前数据

### 3. 统一日志服务
- **LogService**: 提供统一的日志CRUD操作，支持分页查询、条件过滤、导出功能
- **数据清理**: 支持按保留天数自动清理过期日志
- **统计功能**: 提供日志数量统计接口

## 技术实现细节

### AOP切面设计
```java
@Around("@annotation(com.flow.engine.annotation.OpLog)")
public Object around(ProceedingJoinPoint point) throws Throwable {
    // 执行目标方法并捕获异常
    try {
        result = point.proceed();
        return result;
    } catch (Throwable t) {
        error = t;
        throw t;
    } finally {
        recordOperationLog(point, annotation, result, error, startTime);
    }
}
```

### 注解配置选项
- `module`: 业务模块名称
- `operation`: 操作类型（创建/更新/删除等）
- `recordParams`: 是否记录请求参数
- `recordResult`: 是否记录返回结果
- `recordBeforeData/recordAfterData`: 是否记录数据变更前后对比

### 安全考虑
- Token验证失败不影响日志记录
- 敏感信息序列化失败时降级处理
- 日志记录异常被吞掉，确保不影响主业务流程

## 前端集成
Vue3管理端提供完整的日志管理界面：
- 访问日志列表：支持按用户、URL、时间范围筛选
- 操作日志列表：支持按模块、操作类型、用户筛选
- 日志导出功能：支持CSV格式导出
- 日志清理：支持设置保留天数自动清理

## 扩展性设计
- 日志实体可扩展字段
- 支持自定义日志处理器
- 可配置化的日志记录策略
- 异步记录避免阻塞主流程
