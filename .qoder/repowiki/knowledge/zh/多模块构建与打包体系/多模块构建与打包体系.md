## 构建系统概述

该项目采用**多模块独立构建**策略，包含三个独立的子项目：后端 Spring Boot 引擎、Vue3 管理前端、ReactFlow 设计器前端。每个模块使用各自生态的标准构建工具，无统一的顶层构建脚本或 CI/CD 配置。

## 核心构建工具链

### 后端构建（Maven + Spring Boot）
- **构建工具**: Maven，基于 `spring-boot-starter-parent:3.2.0`
- **Java 版本**: JDK 17
- **打包方式**: JAR 包，通过 `spring-boot-maven-plugin` 可执行打包
- **依赖管理**: 集中定义在 `flow-engine/pom.xml`，使用属性变量管理版本号
- **编译配置**: 启用 Lombok 注解处理器，排除运行时依赖

### 前端构建（Vite）
两个前端模块均使用 Vite 5.x 作为构建工具：
- **flow-web (Vue3)**: `vite build` 生产构建，开发服务器端口 3000
- **flow-designer (React)**: `vite build` 生产构建，开发服务器端口 3001
- **开发代理**: 两个前端都配置了 `/api` 代理到 `http://localhost:8080`

## 构建流程与约定

### 启动命令
- **后端**: `mvn spring-boot:run`（自动初始化 SQLite 数据库）
- **前端开发**: `npm run dev`（分别在各模块目录执行）
- **前端构建**: `npm run build`（生成静态资源到 dist 目录）

### 数据库初始化
- 使用 `src/main/resources/db/schema.sql` 在应用启动时自动建表
- 支持 19 张业务表的完整初始化
- 健康检查端点 `/actuator/health` 验证服务状态

### 测试框架
- **后端**: JUnit 5 + Spring Boot Test，覆盖单元测试和集成测试
- **前端**: ESLint 代码检查（仅 flow-designer 模块配置）

## 构建产物结构
- **后端**: `flow-engine/target/flow-engine-1.0.0.jar`
- **前端**: `flow-web/dist/` 和 `flow-designer/dist/` 静态资源
- **数据库**: SQLite 文件 `flow_engine.db`（开发环境）

## 缺失的构建能力

当前项目**缺少以下企业级构建能力**：
- 无 Dockerfile 或 docker-compose 配置文件
- 无 CI/CD 流水线配置（GitHub Actions、Jenkins 等）
- 无统一的构建脚本（Makefile、build.sh 等）
- 无版本发布自动化流程
- 无前后端联合构建脚本
- 无镜像仓库或制品库配置

## 开发者注意事项

1. **环境要求**: 需要 JDK 17+、Node.js 18+、Maven 3.6+
2. **依赖下载**: package-lock.json 指向内部 npm 镜像源，需确保网络可达
3. **端口冲突**: 后端 8080、Vue 前端 3000、React 设计器 3001
4. **数据库隔离**: 每个模块运行会产生独立的 SQLite 数据库文件
5. **构建顺序**: 建议先启动后端服务，再启动前端进行联调
