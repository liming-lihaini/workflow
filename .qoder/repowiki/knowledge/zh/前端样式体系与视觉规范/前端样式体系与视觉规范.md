## 1. 系统与方法论

- **双前端架构**：管理端 `flow-web`（Vue3 + Ant Design Vue）与设计器 `flow-designer`（ReactFlow + React）各自独立构建，通过 Vite 开发。
- **CSS 变量驱动的主题系统**：两个前端项目均使用 `:root` CSS 自定义属性集中定义色板、字体、间距、圆角等设计令牌，禁止在组件中硬编码颜色/尺寸。
- **全局样式覆盖**：通过入口 CSS 对 Ant Design Vue 的按钮、输入框、表格等默认样式进行统一覆写，保证品牌一致性。
- **文档化设计规范**：`docs/front-design.md` 以中文规范形式沉淀了色彩、布局、按钮、卡片、表单、交互等全量规则，作为跨团队约束。

## 2. 关键文件与包

- 设计令牌与全局样式
  - `flow-web/src/styles/variables.css`：管理端主题变量 + 全局基础样式（body、card-wrap、page-header、action-link、ant-btn 覆写等）。
  - `flow-designer/src/index.css`：设计器主题变量 + 画布/工具栏/节点面板/配置面板/消息提示/React Flow 覆盖样式。
- 设计规范文档
  - `docs/front-design.md`：完整的前端视觉与交互规范，涵盖色彩体系、间距、字体、按钮、卡片、登录页、主页、详情页、全局交互及强制约束。
- 依赖声明
  - `flow-web/package.json`：Ant Design Vue 4.x、Pinia、Vue Router、Axios。
  - `flow-designer/package.json`：ReactFlow 生态（core/background/controls/minimap）、React 18、Axios。

## 3. 架构与约定

- **设计令牌分层**
  - 主色 / 辅助状态色 / 文字色 / 背景色 / 边框与圆角 / 间距 全部收敛到 `--color-*`、`--text-*`、`--bg-*`、`--border-*`、`--radius-*`、`--gap-*` 命名空间。
  - 两前端项目的令牌值完全一致，确保管理端与设计器视觉统一。
- **全局样式组织**
  - 所有页面容器使用 `.page-wrap`，内容块使用 `.card-wrap`，操作区使用 `.page-header`，形成统一的页面骨架。
  - 列表操作列统一使用 `.action-link`，危险操作加 `.danger` 变体。
- **组件库样式覆盖策略**
  - 通过选择器直接覆写 Ant Design Vue 的 `.ant-btn`、`.ant-input`、`.ant-table-*` 等类名，保持组件 API 不变的前提下统一外观。
  - 设计器侧通过 `.react-flow__*` 选择器定制连线、手柄、小地图等原生样式。
- **响应式与适配**
  - 规范明确最小宽度 1200px，低于该宽度出现横向滚动条；登录页支持移动端上下布局切换。
- **交互微动效**
  - 按钮点击 `transform: scale(0.98)`，输入框聚焦边框变为 `--color-primary`，Toast 使用 slideIn 动画。

## 4. 开发者应遵循的规则

1. **必须使用 CSS 变量**：任何颜色、字号、行高、圆角、间距都从 `--color-*` / `--text-*` / `--bg-*` / `--radius-*` / `--gap-*` 引用，禁止写死十六进制或 px。
2. **页面结构分层**：面包屑 → 操作栏 → 卡片模块，顺序不可打乱；外层容器用 `.page-wrap`，内容块用 `.card-wrap`。
3. **按钮统一封装**：优先使用 Ant Design Vue 按钮组件并通过全局样式覆盖，不手写原生 button 样式；主/次/危险/文字四种类型严格对应规范。
4. **表格与列表**：表头背景固定为 `#f7f8fa`，行 hover 使用 `--bg-hover`，操作列统一文字按钮并控制间距。
5. **登录页与主页布局**：登录页采用渐变背景 + 居中卡片；主页采用 Header + Sidebar + Main 经典三栏，侧边栏折叠后宽度 64px。
6. **表单弹窗**：标签宽度统一 120px，表单项间距 16px，底部按钮靠右排列（取消 + 保存）。
7. **响应式底线**：最小可视宽度 1200px，低于该宽度允许横向滚动，不做挤压布局。
8. **设计器专属**：节点拖拽、连线、选中态、网关菱形形状、Toast 提示等样式均在 `index.css` 中统一定义，新增节点需复用现有 token。
