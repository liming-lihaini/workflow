- 节点类型以声明式 schema 集中定义在 `nodeTypes.js`，通过 `NODE_TYPES_CONFIG[type].schema` 驱动 NodePalette 展示、FlowNode 渲染与 ConfigPanel 表单生成，避免在各组件中硬编码字段。
- 子组件保持纯展示职责，通过 props 接收回调（如 `onDragStart`、`onNodeConfigChange`、`onEdgeConfigChange`），由 `App.jsx` 持有并更新全局 nodes/edges 状态。
- HTTP 请求统一走 `src/api.js` 的 axios 实例，遵循 `baseURL: '/api/v1'` + 资源路径风格（如 `/process/definitions`），并在请求拦截器自动附加 `Authorization: Bearer <token>`。
- 用户反馈采用本地 `toast` 状态 + 3 秒自动清除的轻量实现，而非引入第三方通知库。
