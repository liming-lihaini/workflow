本模块采用三层职责划分：
- `DataModelController`（REST 层）：暴露 `/api/v1/data-models` 下的模型 CRUD、publish、form-fields 以及模型实例的 create/get/update 接口，统一通过 `Result<T>` 包装响应；模型实例操作委托给同包外的 `ModelInstanceManager`。
- `DataModelService`（业务层）：封装模型生命周期（草稿→已发布）、唯一性校验、版本自增、状态保护（仅草稿可更新/删除），并通过 `toResponse` 将持久化实体 `DataModel` 与解析后的 `DataModelRequest` 组装为对外 DTO；同时生成主表+子表的字段映射列表供前端表单绑定。
- `DataModelParser`（解析层）：集中处理模型 JSON 的 parse/validate/toJson，并实现两类计算字段求值——行内表达式（调用 `ExpressionUtils.eval`）与聚合表达式 `sum(table.field)`（正则识别后遍历子表累加）。
依赖方向单向：Controller → Service → (Mapper + Parser)，Parser 不反向依赖 Service；模型定义以 JSON 字符串形式经 `DataModelParser` 在 `DataModel.modelJson` 中持久化，读写时由 Service 负责序列化/反序列化。
