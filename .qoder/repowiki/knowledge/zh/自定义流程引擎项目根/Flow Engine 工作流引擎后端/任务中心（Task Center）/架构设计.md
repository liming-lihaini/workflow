本模块采用经典的 Controller → Service → Mapper 三层结构：
- `TaskController` 作为 HTTP 入口，统一以 `/api/v1/tasks` 为前缀暴露 REST API，所有响应经 `Result<T>` 包装；除查询接口外，其余写操作均委托给 `TaskService`。
- `TaskService` 是核心业务层，持有 `TaskMapper`（MyBatis-Plus）、`FlowEngine`（流程引擎抽象）和 `CacheManager`（Spring Cache），负责任务状态机转换、权限校验与缓存一致性维护；所有写方法标注 `@Transactional`，并在持久化后调用 `evictTodoCache` / `evictCandidateCache` 按 assignee 或 candidateUsers 维度清理 `task:todo:{userId}` 命名空间下的待办缓存。
- `Task` 实体通过 MyBatis-Plus 的 `@TableName("wf_task")`、`@TableId(AUTO)` 及 `FieldFill.INSERT/INSERT_UPDATE` 与数据库表 `wf_task` 映射，createTime/updateTime 由全局填充器自动注入。
- 依赖方向单向向外：Controller 仅依赖 Service；Service 依赖 Mapper、FlowEngine、CacheManager，不反向依赖 Controller 或 DTO 请求体，DTO（如 `ClaimTaskRequest`、`RejectTaskRequest`、`TaskResponse`）位于上层 `dto` 包中，由 Controller 直接消费。
