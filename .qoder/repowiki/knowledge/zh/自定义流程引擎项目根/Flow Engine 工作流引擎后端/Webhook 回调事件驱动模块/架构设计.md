本模块采用典型的 Controller-Service-Listener 三层结构：
- `WebhookController`（REST 层）：暴露 `/api/v1/webhooks` 下的 CRUD、手动触发 (`/{id}/trigger`)、日志查询与重试等接口，统一通过 `Result<T>` 包装返回；日志查询直接走 `WebhookLogMapper`。
- `WebhookService`（领域服务层）：负责 Webhook 配置的持久化（MyBatis-Plus `WebhookMapper`）、唯一性校验、按 processKey/nodeId/eventType 匹配启用的 Webhook（`findWebhooksByEvent`），并在 DTO 与实体间做 JSON 序列化/反序列化。
- `WebhookEventListener`（事件适配层）：作为 Spring `@Component` 监听 `ProcessStartedEvent` / `NodeEnteredEvent` / `NodeCompletedEvent` / `ProcessCompletedEvent`，根据事件类型构造 payload 后委托 `WebhookScheduler.scheduleAsyncWebhook` 异步执行。
依赖方向单向：Controller → Service + Scheduler；Listener → Service + Scheduler；Service 仅依赖 MyBatis Mapper，不感知事件与 HTTP 调用细节。
