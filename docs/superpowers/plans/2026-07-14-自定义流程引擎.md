# 自定义流程引擎实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 构建一套可自定义的流程引擎，支持可视化流程设计、自定义节点类型、会签/加签等复杂审批模式

**Architecture:** 采用插件式架构，节点即插件，通过JSON配置定义流程，支持Spring Boot自动装配

**Tech Stack:** Java 17, Spring Boot 3.x, MyBatis-Plus, MySQL, Redis

## 2. 数据库表设计

### 2.1 流程定义表 (flow_process_definition)

```sql
CREATE TABLE flow_process_definition (
    id              BIGINT          PRIMARY KEY AUTO_INCREMENT,
    process_key     VARCHAR(64)     NOT NULL COMMENT '流程标识',
    name            VARCHAR(128)    NOT NULL COMMENT '流程名称',
    version         VARCHAR(32)     NOT NULL DEFAULT '1.0' COMMENT '版本号',
    category        VARCHAR(64)     COMMENT '流程分类',
    description     VARCHAR(512)    COMMENT '流程描述',
    process_json    TEXT            NOT NULL COMMENT '流程定义JSON',
    status          TINYINT         NOT NULL DEFAULT 0 COMMENT '状态: 0-未发布, 1-已发布',
    tenant_id       VARCHAR(32)     COMMENT '租户ID',
    create_by       VARCHAR(64)     COMMENT '创建人',
    create_time     DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted         TINYINT         NOT NULL DEFAULT 0,
    UNIQUE KEY uk_process_key_version (process_key, version)
) COMMENT '流程定义表';
```

### 2.2 流程实例表 (flow_process_instance)

```sql
CREATE TABLE flow_process_instance (
    id                      BIGINT          PRIMARY KEY AUTO_INCREMENT,

    process_key             VARCHAR(64)     NOT NULL COMMENT '流程标识',
    process_name            VARCHAR(128)    NOT NULL COMMENT '流程名称',

    instance_no             VARCHAR(64)     NOT NULL COMMENT '实例编号',
    title                   VARCHAR(256)    COMMENT '流程标题',
    status                  TINYINT         NOT NULL DEFAULT 0 COMMENT '状态: 0-运行中, 1-已完成, 2-已终止, 3-已暂停',
    current_node_id         VARCHAR(64)     COMMENT '当前节点ID',
    current_node_name       VARCHAR(128)    COMMENT '当前节点名称',
    initiator               VARCHAR(64)     NOT NULL COMMENT '发起人',
    variables               TEXT            COMMENT '流程变量JSON',
    start_time              DATETIME        NOT NULL COMMENT '开始时间',
    end_time                DATETIME        COMMENT '结束时间',
    duration                BIGINT          COMMENT '流程耗时(毫秒)',

    update_time             DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
```

```java
@Component
public class NodeHandlerRegistry {
```

```java
@Component
public class NodeExecutor {
    
    public void execute(ExecutionContext context) {
        NodeHandler handler = registry.getHandler(context.getNodeType());
        
        // 1. 进入前处理
        handler.onEnter(context);
        
        // 2. 执行节点逻辑
        handler.execute(context);
        
        // 3. 离开后处理
        handler.onLeave(context);
        
        // 4. 推进到下一节点
        moveToNextNode(context);
    }
}
```

### 5.2 会签结果计算

```java
public CounterSignResult calculate(CounterSignGroup group) {
    switch (group.getSignMode()) {
        case ALL_PASS:
            return group.getRejectCount() == 0 ? APPROVED : REJECTED;
        case RATIO:
            double actualRatio = (double) group.getPassCount() / group.getTotalCount();
            return actualRatio >= group.getPassRatio().doubleValue() ? APPROVED : REJECTED;
        case ONE_PASS:
            return group.getPassCount() > 0 ? APPROVED : REJECTED;
        case ONE_REJECT:
            return group.getRejectCount() > 0 ? REJECTED : APPROVED;
        default:
            throw new FlowException("未知的会签模式");
    }
}
```
