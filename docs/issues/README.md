# 自定义流程引擎 — 开发 Issue 清单

> 本清单将《开发计划》（`docs/开发计划.md`）拆分为**可独立运行、可独立验证**的 Issue。
> 每个 Issue 均提供：稳定接口契约、独立运行方式（依赖未就绪时用 Stub/Mock 隔离）、可验证的测试用例（映射到 PRD/TRD 验收点）。
>
> **技术基线（以 TRD 为准）**：Spring Boot 3.x / JDK17 / SQLite 3.x / MyBatis-Plus 3.5.x / Caffeine / Vue3+AntD / React Flow 11.x。
> ⚠️ 与 `docs/superpowers/plans/` 实现计划的技术栈（MySQL+Redis）冲突，全部 Issue 以 TRD 的 SQLite+Caffeine 为准。

## 验收门禁通用规则（所有 Issue 适用）

1. **独立运行**：`mvn spring-boot:run` 可启动；SQLite 表由 `src/main/resources/db/schema.sql` 在启动时自动初始化（spring.sql.init）。
2. **独立验证**：每个 Issue 至少包含 1 个 `@SpringBootTest` 集成测试或 MockMvc 接口测试，不依赖其它 Issue 的未完成代码；上游未就绪时通过 `@MockBean`/Stub 接口隔离。
3. **契约稳定**：本 Issue 对外暴露的 API 路径（TRD §3）/Java 接口签名，在 Issue 关闭后不再破坏性变更。
4. **映射到验收点**：每个 Issue 的“验证用例”逐条对应 PRD §验收标准 或 TRD §对应章节，并标注通过条件。
5. **性能基线**（涉及运行引擎/任务的 Issue）：本地 SQLite 下，单实例创建 < 500ms、节点执行 < 200ms（仅记录基线，M5 统一压测）。
