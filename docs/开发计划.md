## 3. 技术架构与关键约束

### 3.1 分层架构（TRD §1.1）

表现层（Vue3 后台 / React Flow 设计器 / REST API）→ 应用层（流程设计、运行引擎、任务中心、表单、节点插件、监控服务）→ 领域层（流程定义/实例/任务域）→ 基础设施层（SQLite + Caffeine）。

### 3.2 关键技术约束（TRD §5）

- **状态机**：草稿→运行→完成，支持暂停/恢复/终止（终止不可逆）。
- **并发安全**：本地锁 + 数据库乐观锁（非分布式锁，单库 SQLite 场景）。
- **事件驱动**：Spring Event 单节点内分发；多节点通过定时任务轮询。
- **缓存**：Caffeine 缓存流程定义/实例/待办/表单/数据模型（见 TRD §5.4 表）。
- **安全**：JWT + 内存会话；Spring Security 自定义过滤器；敏感字段脱敏；三员权限隔离。

### 3.3 非功能需求基线（PRD §非功能性需求）

| 指标 | 要求 | 验收阶段 |
|------|------|---------|
| 单流程实例创建 | < 500ms | M5 性能测试 |
| 节点执行平均响应 | < 200ms | M5 性能测试 |
| 并发流程实例 | > 1000 | M5 压测 |
| 流程模板数量 | > 10000 | M5 压测 |
| 可用性 | > 99.9%，支持集群部署 | M5 |
| 安全 | 认证授权、加密传输、审计日志、敏感脱敏 | M4 |

## 4. 工作分解结构（WBS）

```
P. 项目准备
  P1 需求与TRD对齐评审
  P2 开发环境与CI/CD搭建

  1.1 项目脚手架(pom/启动类/application.yml)

  2.1 流程定义CRUD(版本/导入导出)

  3.1 流程实例管理(发起/暂停/恢复/终止)

5. 表单引擎
  5.1 表单定义(JSON/组件库)
  5.2 数据绑定与校验

7. 节点插件体系
  7.1 内置节点(start/end/userTask/serviceTask/gateway/scriptTask/subProcess)
  7.2 自定义节点扩展(onEnter/execute/onLeave)
8. 后台管理
  8.1 部门/用户/角色/权限
  8.2 日志(访问/操作)
  8.3 数据字典
  8.4 三员管理(隔离/审计)
9. 流程监控
  9.1 实例监控(轨迹/变量/耗时)
  9.2 手动干预/异常恢复/导出
10. 前端
  10.1 Vue3 管理后台(各模块页面)
  10.2 React Flow 可视化设计器
11. 质量与交付
  11.1 单元/集成测试
  11.2 性能/压测
  11.3 部署(Docker Compose)与文档
```

## 9. 风险管理

| 风险 | 来源 | 等级 | 应对 |
|------|------|------|------|
| 复杂流程性能下降(嵌套子流程/并行网关) | PRD风险1 | 高 | 流程复杂度限制；M5 性能测试先行；Caffeine 缓存热点定义 |
| 数据模型变更影响历史数据 | PRD风险2 | 中 | 字段版本管理；变更兼容评估 |
| 插件版本兼容 | PRD风险3 | 中 | 插件版本校验；保持 NodeHandler API 稳定 |
| 技术栈不一致(MySQL/Redis vs SQLite/Caffeine) | 计划冲突 | 高 | 以 TRD 为准，统一实现计划数据源与缓存 |
| 三员权限边界复杂 | BRD/TRD | 中 | 早期冻结权限矩阵；M4 专项审计测试 |
| 前端设计器(React Flow)与后台(Vue3)技术栈分裂 | TRD | 中 | 设计器以独立子应用/微前端集成，明确契约接口 |
